## Summary of concepts

- Progress does not always come from exploit chains or labs. Sometimes it comes from planning, structure, and environment setup.
- A project becomes much easier to work on once the overall structure is clear, even if no code is written yet.
- Setting up a minimal Linux system from scratch is useful practice because it reinforces:

  - boot setup
  - package management basics
  - working without a GUI

## Deeper understanding (optional but recommended)

One useful lesson from today is that **infrastructure and setup are part of technical growth too**.

Installing a minimal Arch VM is not the same as solving a box, but it still strengthens system understanding:
- disk layout
- boot process
- working in a minimal environment
- package management with `pacman`
- thinking about what a system actually needs to become usable

That kind of setup work is slower and less flashy, but it builds confidence around Linux systems in a way that prebuilt environments do not.
